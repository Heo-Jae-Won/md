## <span style="color:#802548">_thread란?_</span>
- 대부분의 OS는 멀티태스킹을 지원한다.
- CPU의 코어 하나 당 하나의 작업만 수행할 수 있으므로, 작업의 개수는 코어의 개수와 일치한다.
- 각 코어는 여러 작업을 번갈아 가며 수행하면서 여러 작업이 동시에 이뤄지는 것처럼 보이게 한다.
- 메신저로 채팅하면서 파일을 다운로드 받으면서 대화를 할 수 있는 이유가 바로 멀티쓰레드이기 때문이다.
- 싱글스레드로는 사용자의 요청마다 새로운 프로세스를 생성하여 더 많은 시간과 메모리 공간이 필요하다.
- 따라서 멀티스레드를 사용하는 것이다. 스레드끼리는 자원을 공유할 수 있다. 
- 스레드끼리라고 해도 call stack은 공유하지 않는다. 각자 독립 작업을 수행하는 데 자신만의 call stack이 필요하기 때문이다.
- 그러나 멀티 스레드가 늘 좋은 것은 아니다. 쓰레드 간 작업전환(context switching)에 드는 시간이 꽤나 걸린다.

- Thread를 구현하는 방법에는 Thread class를 extends하거나 Runnable interface를 구현하는 법이 있다.

```java
class MyThread extends Thread{
    public void run()
}

MyThread t1 = new MyThread();

t1.start();
```

```java
class MyThread implements Runnable{
    public void run()
}

Runnable r = new MyThread();
Thread t2 = new Thread(r);

//Thread t2 = new Thread(new MyThread());
t2.start(); 
```

-  t2.run()을 하게 되면 run method가 기존 thread의 call stack에 쌓일 뿐이다.
-  따라서 t2.start();로 해야 새로운 thread에 call stack으로 run method가 들어가게 된다.
-  또한 thread instance는 한 번 실행시키면 다시 실행시킬 수 없다. 반드시 새로운 instance를 만든 뒤에 start가 필요하다.

```java
public static void main(String[] args){
    MyThread t1 = new MyThread();
    t1.start();
    t1.start(); //error
}

```

```java
public static void main(String[] args){
    MyThread t1 = new MyThread();
    t1.start();
    t1 = new MyThread();
    t1.start(); //ok
}
```

- thread를 main thread 이외에 하나 더 만들게 되면 main이 종료되면 프로그램이 종료되는 게 아니다.
- 모든 사용자 thread가 실행되지 않고 있을 때 프로그램이 종료된다.

- main thread의 main method가 실행이 끝나고나서 새로운 thread가 시작된다.
- 한 쓰레드가 예외가 발생해서 종료되어도 다른 쓰레드의 실행에는 영향을 미치지 않는다.
- 따라서 main thread가 아닌 다른 thread에서 exception이 났어도 영향이 없다.

- 멀티코어에서는 멀티 스레드로 작업하게 되면, 작업끼리 서로 겹치게 될 수 있다.
- 예를 들어 console이라는 자원을 놓고 두 쓰레드가 경쟁을 하게 될 수 있다. 그러면 싱글 코어의 싱글 쓰레드보다 더 느려진다.
- 또한 쓰레드는 OS의 프로세스 스케줄러의 영향을 받기 때문에 매번 경과시간이 차이가 있다.
- 두 쓰레드가 서로 다른 자원을 사용한는 작업이라면 멀티 쓰레드가 더 효율적이다. 
- 데이터를 입력받기- 네트워크로 파일을 주고받기 - 프린터로 파일을 출력하기와 같은 외부기기와의 입출력을 필요로 하는 경우가 그러하다.

- 두개의 쓰레드로 두개의 작업을 하게 되는 경우, 하나의 쓰레드가 두 개의 작업을 수행하는 것보다 더 걸릴 수도 있다.
- 그 이유는 쓰레드간의 작업전환에 시간이 걸리기 때문이다.
- 작업 전환 시에는 다음에 실행해야 할 위치, 정보를 저장하고 읽어오는 시간이 소요된다.

- 여러 쓰레드가 여러 작업을 동시에 진행하는 것은 병행(concurrent)
- 하나의 작업을 여러 쓰레드가 나눠 처리하는 것은 병렬(parallel)

## <span style="color:#802548">_thread의 우선순위란?_</span>
- 쓰레드가 가질 수 있는 우선순위는 1부터 10까지다.

```java
void setPriority(int newPriority)
int getPriority()

public static final int MAX_PRIORITY = 10;
public static final int MIN_PRIORITY = 1;
public static final int NORM_PRIORITY = 5;
```
- main method에서 나온 것들은 기본이 5이다.
- 하지만 문제는 우선순위를 앞에 놓는다고해서 늘 우선적으로 먼저 '모두' 처리되지는 않는다
- 우선순위 자체가 먼저 처리되는 것이 아니라, 쓰레드가 얻는 실행시간을 늘리느냐 줄이느냐의 차이이기 때문이며,
- 현대는 대부분의 CPU가 멀티코어기 때문에, 번갈아가면서 작업을 하기 때문이다.
- 따라서 JVM의 구현방식, OS의 스케줄링을 확인해서 실제로 우선순위를 높이면 높인 것 먼저 모두 실행하는 지 봐야한다.
- 우선순위를 통해 먼저 실행시키고 완성시킨다는 개념은 포기하는게 좋다. 실제로 우선순위 개념은 잘 쓰이지 않는다.
- 확실한 방법은 오히려 thread에 우선순위를 부여하는 게 아니라 PriorityQueue에 저장하여 우선순위가 높은 작업 순으로 처리하는 것이다.
  
```java

import java.util.PriorityQueue;

class Job implements Comparable<Job> {
    private int priority;
    private String description;

    public Job(int priority, String description) {
        this.priority = priority;
        this.description = description;
        System.out.println("New job: " + description);
    }

    public int getPriority() {
        return priority;
    }

    public String getDescription() {
        return description;
    }

    @Override
    public int compareTo(Job other) {
        return Integer.compare(this.priority, other.priority);
    }
}

class Worker implements Runnable {
    private PriorityQueue<Job> queue;

    public Worker(PriorityQueue<Job> queue) {
        this.queue = queue;
    }

    @Override
    public void run() {
        while (true) {
            Job job = queue.poll();
            if (job == null) {
                break;
            }
            System.out.println("Processing job: " + job.getDescription());
        }
    }
}

public class Main {
    public static void main(String[] args) {
        PriorityQueue<Job> queue = new PriorityQueue<>();

        // Start worker threads
        int numThreads = 3;
        Thread[] threads = new Thread[numThreads];
        for (int i = 0; i < numThreads; i++) {
            threads[i] = new Thread(new Worker(queue));
            threads[i].start();
        }

        // Add jobs to the queue
        queue.offer(new Job(3, "Low priority job"));
        queue.offer(new Job(10, "High priority job"));
        queue.offer(new Job(1, "Very high priority job"));

        // Wait for all jobs to finish
        for (Thread thread : threads) {
            try {
                thread.join();
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}
```

- 쓰레드는 반드시 어느 하나의 쓰레드 그룹에 속해야만 한다. 그래야 다른 쓰레드에서 의도치 않게 해당 쓰레드를 변경하지 못하기 때문이다.
- 쓰레드 그룹을 정하지 않는다면 자신을 생성한 쓰레드 그룹에 속하게 된다. 쓰레드 그룹 안에 하위 쓰레드 그룹이 있을 수도 있다. 폴더처럼 말이다.
- JVM은 main과 system이라는 쓰레드 그룹을 만들고 JVM운영에 필요한 method를 포함시킨다.
- main method는 main thread group에, GC의 finalizer method는 system thread group에 속한다.
- 우리가 생성하는 모든 thread group은 main thread group의 하위에 속한다. 
- 따라서 Thread그룹을 별도로 지정하지 않으면 main thread group에 속하게 된다.

- 개별 쓰레드가 아니라 쓰레드 그룹의 우선순위도 존재한다.

## <span style="color:#802548">_daemon thread란?_</span>
- daemon thread란 다른 일반 쓰레드의 작업을 돕는 보조적인 역할을 수행한다. 
- daemon 쓰레드는 일반 쓰레드가 실행이 끝나서 종료되면 같이 종료된다.
- 데몬 thread는 대기 중에 있다가 특정 조건이 만족되면 작업을 수행하고 다시 대기하게끔 소스코드를 짜면 된다.
- 데몬 쓰레드도 코딩방식은 동일하다. 다만 실행하기 전에 setDaemon(true)만 호출해주면 된다.
- 참고로 데몬 쓰레드가 생성한 thread는 모두 daemon thread가 된다.

## <span style="color:#802548">_thread의 실행제어 method란?_</span>
- 앞서서 말한 우선순위는 스케줄링에 관한 것이다.
- 그럼 이제는 동기화를 해줘야 한다.
- 동기화를 하는 Thread의 method는 아래와 같다.

```java
sleep();        //지정시간만큼 일시정지 후 실행대기
join();         //지정시간만큼 일시정지 후 다시 원래 thread 실행
interrupt();    //일시정지 thread를 꺠워 실행대기상태로 변경 or 실행중인 thread에 flag를 주어 thread 종료시키기
stop();         //즉시종료. 쓰지 말것. boolean 변수로 통제가 더 좋음
suspend();      //일지정지. 쓰지말것. boolean 변수로 통제가 더 좋음
resume();       //suspend()로 멈춘 thread를 깨워 실행대기상태로 변경. 쓰지 말것
yield();        //주어진 실행시간을 다른 thread에게 양보하고 실행대기상태로 변경
```

## <span style="color:#802548">_thread의 동기화란?_</span>
- 멀티쓰레딩 환경에서는 프로세스 내에서 자원을 공유한다.
- 따라서 쓰레드 A가 작업을 하다가, B로 넘겼을 때 B가 데이터를 바꾸면 A는 바뀐 데이터가 출력된다.
- 따라서 한 쓰레드가 작업을 마치기 전까지는 다른 쓰레드가 데이터를 건들지 못하게 해야한다.
- 그러한 작업을 동기화라고 하며, 그를 위해 임계 영역(critical section)과 lock이라는 기술이 존재한다.
- 임계영역은 보통 공유 데이터를 사용하는 코드영역으로 지정하며, 공유 데이터가 가지고 있는 lock을 획득한 단 하나의 쓰레드만 코드를 수행할 수 있게 한다.
- 동기화는 아래와 같이 진행할 수 있다. 가능하면 method 전체가 아니라 동기화할 부분에만 synchronized를 거는 게 좋다.

# fork & join
- 하나의 작업을 여러 쓰레드가 동시에 처리하는 작업은 fork & join api를 이용한다.
- ForkJoinPool은 fork & join 프레임워크에서 제공하는 Thread pool로, 지정된 수의 쓰레드를 새엉해서 미리 만들어놓고 반복해서 재사용한다.
- 쓰레드 풀은 쓰레드가 수행해야 하는 작업이 담긴 큐를 제공하며, 각 쓰레드는 자신의 작업 큐에 담긴 작업을 순서대로 처리한다.
- 쓰레드 풀은 기본적으로 코어의 개수와 동일한 개수의 쓰레드를 생성한다.
```java

    SumTask(long from, long to) {

        this.to = to;

       long size = to - from + 1;

    if (size <= 5) {

    SumTask rigthSum = new SumTask(half + 1, to);   //0부터 10이 들어오면, size가 큼.. 따라서 half로 나눠서 다시 0-5는 fork를 뜸..즉 다른 thread가 가져감.
                                                    //남은 것은 6-10이며 size가 5이므로 sum을 실행함. return한 값으로 종료됨. 그 아래로 가지 않음.
                                                    //남은 0-5는 size가 5보다 크므로 sum함수를 실행하지 않고 다시 SumTask를 만듦. 0-2, 3-5로 쪼개짐(0-2가 fork)
                                                    //0-2는 size가 5이하이므로 sum을 반환함. 3임. 
                                                    //3-5는 size가 5이하이므로 sum을 반환함. 12임.

    return rigthSum.compute() + leftSum.join();
    }
```

```java
		FolderProcessor documents = new FolderProcessor("C:\\Documents And Settings", "log");

		pool.execute(system);
		pool.execute(apps);

				for (File current : content) {
					if (current.isDirectory()) {
						FolderProcessor task = new FolderProcessor(current.getAbsolutePath(), extension);
						task.fork();
						tasks.add(task);
					} else {
						if (checkFile(current.getName())) {
							list.add(current.getAbsolutePath());
						}
					}
				}
			}
			if (tasks.size() > 50) {
				System.out.printf("%s: %d tasks ran.\n", file.getAbsolutePath(), tasks.size());
			}
			addResultsFromTasks(list, tasks);
			return list;
		}

		private void addResultsFromTasks(List<String> list, List<FolderProcessor> tasks) {
			for (FolderProcessor item : tasks) {
				list.addAll(item.join());
			}
		}

		private boolean checkFile(String name) {
			return name.endsWith(extension);
		}
	}
```
